# 제품 맥락

## 사용자·시나리오

현장 운영자가 **브라우저 한 화면**에서 매트릭스를 다룬다.

1. 장비 연결 상태 확인  
2. Output별 Input 연결 상태 확인  
3. Input/Output **이름** 관리 (현장 라벨)  
4. **수동 라우팅** (Input → Output)  
5. **프리셋** 생성·수정·삭제·순서 변경·실행 (필요 시 실행 전 확인)  
6. **최근 작업** 열람  
7. 선택적 **되돌리기** (undoable 작업만)  
8. 설정 저장·불러오기 (Serial + I/O 이름 등)

## 비즈니스 규칙

- 라우팅 명령 형식: `{input_no}X{output_no}.` (명세 §4.6)
- 프리셋 내 라우팅은 **순차** 실행, 명령 간 **100~200ms** delay 가능; **중간 실패 시 중단**이 초기 기본 (§4.8)
- 되돌리기: `before_routes` 기준으로 동일 Queue 경로로 복원 후 `Status.` (§4.10)
- 연결 테스트: 우선 `Status.`, 보조 `%Version;` (§4.2)
- 표시: 이름 없으면 `Input n` / `Output n`만 (§4.5)

기능 상세·화면 구역·API·JSON 예시는 **`SPEC_A1616HD.md`**를 본문으로 한다.
