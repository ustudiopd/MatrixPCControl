Serial 설정

연결 테스트는 probe_command(기본 .) 한 번 보내고 응답으로 Serial 연결을 확인합니다. 라우팅은 {input}X{output}. 형식입니다.

수동 라우팅

장비에서 확인된 형식: — (플레이스홀더 input, output)

테스트

라우팅 상태 (Output → Input)

수동 라우팅 성공 시 장비 응답에 Ch:0,V:01 형태(16줄)가 있으면 전체 출력을 그 값으로 갱신합니다. 없으면 방금 보낸 출력 한 칸만 반영합니다.

마지막 조회: —

출력연결된 입력
마지막 조회 raw 일부

Input / Output 이름

라우팅 표에 표시되는 이름입니다. data/config.json에 저장됩니다.

프리셋

data/presets.json — 실행 시 Serial은 한 번에 순차 전송됩니다. 되돌리기 스택에도 쌓입니다.

새 프리셋

최근 작업

연결 테스트·수동 라우팅·프리셋·되돌리기 기록 (최대 50건, data/history.json).

되돌리기

성공한 수동 라우팅·프리셋·되돌리기 작업이 스택에 쌓입니다 (최대 20건, data/undo_stack.json). 되돌리기 후에는 되돌리기 취소(재적용)용 항목이 이어서 쌓여, 다시 직전 라우팅으로 돌릴 수 있습니다. 이전 입력이 없으면 Serial 없이 표시만 복원합니다.
